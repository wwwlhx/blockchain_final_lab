## 当前架构

```text
React + TypeScript
        |
Express + TypeScript ----- SQLite
        |
ethers.js + JSON-RPC
        |
Solidity AssetRegistry ----- Hardhat
```

## 数据一致性

Hardhat 本地链每次重新启动后需要重新部署合约。后端会读取部署文件中的网络、合约地址和部署时间，检测到新部署时自动清理上一轮演示的 SQLite 资产索引，防止链上为空但仪表盘仍显示旧数据。

## 已知边界

- 本地 Hardhat 节点不提供生产级持久化。
- 元数据目前保存在本地，可继续扩展到 IPFS。
- MetaMask 尚未用于登记、转移和撤销交易的用户签名。
- 当前没有登录、审核和复杂角色权限。
- SQLite 和同步策略面向单机课程演示。

## 答辩定位

本项目应描述为：

> 面向数字资产的可验证技术证据链原型，通过文件指纹、声明指纹和链上权属记录，实现完整性校验与生命周期追溯。

不应描述为：

- 自动完成法律意义上的版权认定
- 所有交易均由 MetaMask 用户签名
- 已使用 IPFS 或公链生产网络
- 完全去中心化的商业系统

详细说明见 [README.md](README.md)，演示顺序见 [docs/demo-script.md](docs/demo-script.md)。
